# 邊直走邊轉問題修復 - 駕駛員/測試員指南

## 簡單版本（適合快速理解）

### 問題是什麼？
- ❌ 按住左搖桿直走 + 右搖桿轉向時
- ❌ 機器人在原地轉圈（像陀螺儀）
- ❌ 而不是邊走邊轉（像汽車的轉彎）

### 原因是什麼？
轉向馬達的大腦（PID 控制器）反應太慢了，跟不上直走命令需要的轉向速度。

### 修復了什麼？
給轉向馬達的大腦「吃了興奮劑」：
- ✅ 反應速度提高 40-50%
- ✅ 轉向更精確
- ✅ 死區變小（轉向更靈敏）

## 安裝和測試

### 1️⃣ 安裝新代碼
**選項 A（推薦）**：
```
1. 用 USB 線連接中樞
2. 打開 Android Studio
3. 點擊 ▶️ Run（綠色播放按鈕）
4. 等待自動編譯和安裝
```

**選項 B（命令行）**：
```powershell
# 打開 PowerShell，切到項目目錄
cd D:\FTC\FTC_2025-2026\Swerve
./gradlew build
./gradlew installDebug
```

### 2️⃣ 立即測試
1. 機器人上電
2. 選擇 **Swerve_Control**
3. 按 **PLAY** 運行
4. 測試：
   ```
   左搖桿 → 前進
   同時 → 右搖桿向左或向右轉
   
   ✓ 如果機器人沿著曲線前進 → 成功！
   ✗ 如果機器人還在原地轉圈 → 需要進一步調試
   ```

## 如果還有問題

### 症狀 1: 仍然在原地轉圈
```
1. 關閉機器人
2. 等 10 秒
3. 重新上電
4. 再試一次（有時系統需要完整重啟）
```

### 症狀 2: 轉向有抖動或過度反應
```
打開 FTC Dashboard：
1. 手機連接到機器人的 WiFi
2. 在瀏覽器打開 192.168.43.1:8080
3. 在 TuningConfig 面板中：
   - 將 _1a_turningP 從 0.8 改為 0.6 或 0.7
   - 將 _1c_turningD 從 0.05 改為 0.1 或 0.15
4. 立即測試（無需重新編譯）
5. 調整直到滿意為止
```

### 症狀 3: 某個輪子轉向明顯慢
```
1. 檢查那個輪子的馬達是否有故障
2. 檢查 USB 線是否正確連接
3. 運行 _1b_TurningEncoderReverse.java 測試轉向方向
```

## 新參數說明（給高級用戶）

| 參數 | 舊值 | 新值 | 作用 |
|------|------|------|------|
| kPTurning | 0.5 | 0.8 | 轉向更快 |
| kITurning | 0.0 | 0.01 | 消除誤差 |
| kDTurning | 0.0 | 0.05 | 減少抖動 |
| kTurningDeadbandDeg | 1.0° | 0.5° | 轉向更靈敏 |

## 什麼時候需要回滾？

若新設定導致嚴重問題（例如：轉向馬達失控），可以回滾：

1. 打開 `Constants.java`（在 Android Studio 中）
2. 找到第 36-46 行，改回舊值
3. 重新編譯部署
