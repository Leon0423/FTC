1Swerve Drive 使用的是剛體逆運動學（Inverse Kinematics）模型。
2將底盤的整體運動 (Vx, Vy, ω) 分解為每個輪子各自需要的速度與角度。
3核心思想：每個輪子的速度向量 = 底盤平移速度 + 該輪繞底盤中心旋轉的切線速度。
4底盤有 3 個自由度（Vx, Vy, ω），4 個模組各有 2 個自由度（speed, angle），系統為過約束——透過 normalize 確保物理可行。

設輪子 i 相對底盤中心的位置為 (x_i, y_i)，底盤速度為 (Vx, Vy, ω)：

Vx_i = Vx − ω × y_i （旋轉切線分量：x 方向） Vy_i = Vy + ω × x_i （旋轉切線分量：y 方向）

這來自剛體運動學：任何一點的速度 = 質心平移速度 + 角速度 × 位置向量的叉積。

矩陣形式：

┌ Vx_i ┐ ┌ 1 0 −y_i ┐ ┌ Vx ┐ │ │ = │ │ │ │ └ Vy_i ┘ └ 0 1 +x_i ┘ │ Vy │ └ ω ┘

四個輪子合起來就是一個 8×3 的矩陣乘以 3×1 的向量，得到 8×1 的結果。

得到各輪的 Vx_i, Vy_i 後，轉換為極座標：

speed_i = √(Vx_i² + Vy_i²) angle_i = atan2(Vy_i, Vx_i)
1Normalize（歸一化輪速）：如果任何一個輪速超過最大物理速度，則等比例縮小所有輪速，保持速度比例不變。
if max(speed_i) > maxSpeed: scale = maxSpeed / max(speed_i) speed_i *= scale (for all i)

這確保不會要求任何馬達超出能力，同時維持各輪速度的相對比例，保證運動方向正確。

2Optimize（最短路徑優化）：如果目標角與當前角差 > 90°，將角度翻轉 180° 並反轉速度方向。這樣輪子最多只需轉 90°。
delta = target_angle − current_angle if |delta| > 90°: target_angle += 180° speed = −speed

原理：輪子轉 180° + 正轉，等效於不轉 + 反轉。選擇轉角較小的方案，提升響應速度。

程式碼流水線對應：

1SwerveJoystickCmd.execute() — 讀搖桿 → deadband → SlewRateLimiter 平滑 → 組成 ChassisSpeeds
2kDriveKinematics.toSwerveModuleStates(chassisSpeeds) — FTCLib 內部執行 8×3 矩陣乘法
3setModuleStates() — powerScale 補償 → normalizeWheelSpeeds()
4SwerveModule.setDesiredState() — optimize() 最短路徑 → Drive 設功率 → Turning PID 追角度

本專案的 kDriveKinematics 輪位定義（kWheelBase = kTrackWidth = 0.22m）：

FL: (+0.11, −0.11) FR: (+0.11, +0.11) BL: (−0.11, −0.11) BR: (−0.11, +0.11) （單位：公尺）

Field-Oriented 模式下，ChassisSpeeds.fromFieldRelativeSpeeds() 會先將搖桿向量旋轉 −θ（θ = 機器人朝向），使輸入從場地座標轉換到機器人座標後，再送入逆運動學。
